# 后台页面游戏筛选分类

本文档按「是否展示游戏筛选器」及「是否允许选择『全部』」对后台页面进行分类，并说明跨页面跳转时筛选状态的保留规则。

> 头部筛选器统一显示为「游戏：」下拉选择框。

---

## 一、页面分类

### 类型 A：全局页面（无游戏筛选）

不展示游戏筛选器，页面数据不依赖游戏维度。

| 模块 | 页面 | 路径 |
|------|------|------|
| 首页 | 工作台 | `/` |
| 论坛资源 | 论坛 | `/forum/list` |
| 内容 | 帖子 | `/content` |
| 内容 | 话题 | `/content/topics` |
| 用户 | 用户列表 | `/users` |
| 用户 | 马甲号 | `/users/sockpuppet` |
| 平台管理 | 平台管理 | `/platform` |
| 权限管理 | 权限管理 | `/permissions` |

### 类型 B：游戏筛选 · 可选（有「全部」选项）

展示游戏筛选器，选择「全部」时展示所有游戏的数据；选择具体游戏时按该游戏筛选。

| 模块 | 页面 | 路径 |
|------|------|------|
| 用户 | 认证账号 | `/users/certified` |
| 创作者管理 | 认证创作者 | `/creator/creators` |
| 创作者管理 | 返利管理 | `/creator/rebate` |
| 创作者管理 | 素材库管理 | `/creator/materials` |
| 创作者管理 | 多语言审校管理 | `/creator/review` |
| 内容 | 官方内容 | `/forum/official` |

### 类型 C：游戏筛选 · 必选（无「全部」选项）

展示游戏筛选器，**无「全部」选项**；`forumId` 为空时自动选中列表中的第一个游戏。

| 模块 | 页面 | 路径 |
|------|------|------|
| 论坛资源 | 社区导航 | `/forum/community-nav` |
| 论坛资源 | 金刚位 | `/forum/diamond` |
| 论坛资源 | 置顶 | `/forum/announcement` |
| 论坛资源 | 分区 | `/tab-route` |
| 论坛资源 | 集合页 | `/collection-pages` |
| 论坛资源 | Wiki 管理 | `/wiki` |
| 论坛资源 | Wiki 配置 | `/wiki/config/[key]` |
| 论坛资源 | 下载按钮 | `/download-button` |

## 二、跳转时筛选状态规则

### 基本原则

`forumId` 存储在全局 Context 中，**跨页面导航时始终保持，不会因跳转而清空**。页面只读取、不重置这个状态。

### 跳转场景矩阵

| 来源页面类型 | `forumId` 状态 | 目标页面类型 | 筛选器显示 | 页面内容展示 |
|-------------|--------------|-------------|-----------|------------|
| 任意 | 任意 | **A（无筛选）** | 隐藏 | 正常展示全部数据 |
| 任意 | 空（全部） | **B（可选）** | 显示，选中「全部」 | 正常展示全部数据 |
| 任意 | 有值 | **B（可选）** | 显示，选中该游戏 | 按该游戏筛选展示 |
| 任意 | 空（全部） | **C（必选）** | 自动选中第一个游戏 | 按第一个游戏展示内容 |
| 任意 | 有值 | **C（必选）** | 显示，选中该游戏 | 按该游戏筛选展示 |

### 典型跳转场景说明

**场景 1：在 C 类页面选了游戏 → 跳到另一个 C 类页面**
- `forumId` 有值，直接带入，新页面正常展示该游戏数据 ✅

**场景 2：在 C 类页面选了游戏 → 跳到 B 类页面**
- `forumId` 有值，带入 B 类页面，按该游戏筛选展示 ✅
- 用户可在 B 类页面切换为「全部」

**场景 3：在 B 类页面选了「全部」→ 跳到 C 类页面**
- `forumId` 为空，`ForumSelectRequired` 自动选中第一个游戏
- 页面直接展示该游戏数据，筛选器同步更新为第一个游戏 ✅

**场景 4：在 A 类页面 → 跳到 C 类页面**
- `forumId` 保持上次离开有筛选页面时的值
- 若上次有选游戏：直接展示内容 ✅
- 若上次是空：自动选中第一个游戏并展示 ✅

**场景 5：在 A 类页面 → 跳到 B 类页面**
- `forumId` 保持上次离开有筛选页面时的值，正常恢复 ✅

## 三、技术实现

- **筛选器展示**：由 `Header.tsx` 的 `FILTER_OPTIONAL_PATHS` / `FILTER_REQUIRED_PATHS` 控制，通过 `useForumFilterMode()` 返回 `'optional' | 'required' | 'none'`
- **状态持久化**：`forumId` 存储在 `ForumFilterContext`，全局唯一，页面跳转不重置
- **必选模式**：`filterMode === 'required'` 时，Select 不渲染「全部」选项，`allowClear` 为 `false`，placeholder 为「请选择游戏」
- **可选模式**：`filterMode === 'optional'` 时，Select 包含「全部」选项，`allowClear` 为 `true`
- **必选自动选中**：由 `ForumSelectRequired` 组件包裹，当 `forumId` 为空时通过 `useEffect` 自动将其设为 `forumOptions[0].id`
- **可选筛选**：页面内使用 `useForumFilter().forumId` 做数据过滤，空时不过滤
