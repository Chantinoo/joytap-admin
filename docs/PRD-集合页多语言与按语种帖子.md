# PRD：集合页多语言、按语种帖子与隐藏（简版）

## 1. 背景与目标

集合页在后台支持**多语言名称**与**各语种独立前台路径**。业务要求：

- **不同语种下的帖子列表互相独立**（按语种分桶），运营按语种分别配置。
- **移除某语种的展示名称时，不删除该语种下已配置的帖子**，数据保留，仅前台不再以该语种名称展示（或整页策略由前台约定）。
- 管理端切换语种时，列表与主标题需让用户**一眼区分「已命名 / 未命名」**，避免误以为无数据。

**目标**：后台集合页具备按语种帖子、隐藏开关、即时保存与「未命名语种」提示能力；文档与 mock 可供联调/演示。

## 2. 数据与兼容

| 项 | 说明 |
|----|------|
| `articlesByLocale` | `Partial<Record<LangCode, Article[]>>`，各语种帖子独立。 |
| `nameI18n` | 各语种展示名称；某语种无条目或为空即视为**未配置该语种展示名称**（`zh` 另认可列表主名称 `name`）。 |
| `link` / `linkI18n` | `link` 为列表与分区模块中的**主链接**（按 `zh → zh-tw → en → …` 取第一个有值的路径）；`linkI18n` 为各语种前台路径。未填某语种路径时，前台可约定回退到主链接。新建时默认写入 **简体中文** 路径（见下节），其它语种在「编辑」中填写。 |
| `hidden` | `true` 时前台不展示该集合页；后台仍可管理；分区里**新增/替换**集合页时下拉不出现该项。 |
| 旧数据 | 仅存在顶层 `articles` 时，读取时合并为默认语种（如 `zh`），写入按语种后去掉旧字段。 |

### 2.1 前台路径约定（与 `LangCode` 对齐）

- **简体中文 `zh`**：`/zh/collection/<序号>`
- **繁体中文 `zh-tw`**：`/zh-tw/collection/<序号>`
- **其它语种**：`/<语种段>/collection/<序号>`，例如 `en` → `/en/collection/6`，`ko` → `/ko/collection/6`
- 新建集合页时系统自动分配下一个 `<序号>`（扫描已有路径中的 `/collection/<n>`），并生成 **默认简体路径** `/zh/collection/N`，同时写入 `link` 与 `linkI18n.zh`。
- 历史裸路径 `/collection/N` 若仍存在数据中，后台仍可展示与匹配；运营可按需在「编辑」中改为带语种前缀的路径。

## 3. 集合页列表

- **帖子数**：展示各语种**合计**（仅数字 + 图标，**不**在格内追加「多语种」等提示文案）；悬停 Tooltip 展示分语种条数（如 `zh：5`）。
- **隐藏**：开关打开 = 隐藏；切换后立即落库并提示。
- **有未命名语种**：若存在「该语种**有帖子**但**无展示名称**」，名称旁显示橙色标签 **「有未命名语种」**；Tooltip 文案：**「部分语种下已有帖子，但未填写该语种的展示名称」**（仅此一句）。
- **多语言名称**：名称旁图标（Popover）查看已配置语种名称；**隐藏**列：开关即时落库。
- **链接列**：展示**主链接**（可点击外链）；悬停 Tooltip 展示各已填语种路径（`code：path`）。分区模块里配置的集合页 `link` 与**任一语种路径**一致即可命中同一集合页（用于「管理帖子」跳转）。

## 4. 管理帖子（集合页详情）

- **语种切换**：Segmented；可选语种 = 已配置展示名称 **或** 该语种下已有帖子。
- **未命名语种**（有帖、无展示名称）：
  - Tab 上显示 **「未命名」** 小标签，语种文案可略灰。
  - 主标题（及面包屑末级）显示 **「未命名」**（橙色），**不再**回退到中文主名称。
  - **不再**在副标题重复橙色短句（避免与 Alert 重复）。
  - 保留 **警告 Alert**（可关闭、`warning`、带图标）：单行文案——**「该语种未填写展示名称，前台将不会展示本语种集合页」**。
- **帖子操作**：新增 / 编辑 / 删除后**立即写入**当前语种数据，**无需**顶部二次「保存」；成功提示使用 `message`（如 **「添加成功」** 等）。
- **切换语种**：若当前语种列表存在未保存的本地修改（极端情况），切换前确认是否丢弃（保留原防护即可）。

## 5. 编辑集合页弹窗（列表「编辑」）

- 标题：**「编辑集合页」**。
- 顶部浅蓝说明区：名称可 **AI 翻译**（选源语言后一键填充其它语种**名称**）；链接需**按行手填**。
- **「\* 集合页配置」**（必填语义）：动态行列表，每行包含——**语种下拉**、**展示名称**、**前台路径（链接）**、行末**删除**（至少保留一行）。语种不可在行间重复。
- **「+ 添加语种」**：追加一行，语种为当前尚未使用的下一项（受 `LANGUAGES` 列表约束）。
- 底部按钮：**「取消」**、**「确定」**（保存）。
- 校验：至少一行非空名称、至少一行非空路径；保存时若**去掉**某语种展示名称且该语种**仍有帖子**：成功提示中说明**帖子仍保留**，可到管理帖子查看。
- **不**在保存时删除该语种 `articlesByLocale` 数据。

## 6. 分区管理（Tab 编辑）

- 可选集合页下拉：**排除** `hidden` 的集合页；选项可展示**帖子总数**（全语种合计）便于识别。

## 7. 演示用 Mock（本地）

- **武器攻略**（示例集合页）：`nameI18n` **不含** `ko`，但 `articlesByLocale.ko` 含帖子，用于演示 **「有未命名语种」** 全流程；`linkI18n` 含 `en`/`ko` 等独立路径示例。

## 8. 非本期 / 待前台

- 前台按用户语言拉取对应 `articlesByLocale`、对应 `linkI18n` 路径（或回退策略）/ 过滤 `hidden` / 未命名语种展示策略。

## 9. 角色

- **运营 / 内容**：列表配置名称、各语种链接、隐藏、多语言；详情按语种管帖。
- **开发**：类型、Context、列表/详情 UI、mock、与前台契约。

## 10. 上线发布（joytap-admin）

- 源码托管于 **GitHub**，默认分支 **`main`**；已完成的功能提交应推送到 `main`。
- **生产环境**：由团队在 **Vercel**（或等价 CI/CD）绑定该仓库后，在 `main` 更新时自动触发 **Production** 构建与部署；亦可本地安装 [Vercel CLI](https://vercel.com/docs/cli) 并登录后执行 `vercel deploy --prod` 手动发布。
- 发布前建议本地执行 `npm run build` 通过后再合并/推送。
